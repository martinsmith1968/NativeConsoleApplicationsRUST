## Implementation Details

### CLI Flag Addition
- Added `--csharp` / `-c` boolean flag to Args struct with `short` and `long` attributes
- Flag triggers C# format translation mode before standard formatting

### Translation Function
- Implemented `translate_csharp_format(format_str: &str) -> Result<String, String>` as pure preprocessing step
- Processes format string before `apply_format()` call, maintaining existing signature integrity
- Lazy regex compilation for efficiency in repeated calls

### Format Specifier Support
- **Passthrough:** `{0}`, `{1}`, `{0:G}`, `{0:g}`
- **Alignment:** `{0,-N}` → `{0:<N}` (left), `{0,N}` → `{0:>N}` (right)
- **Numeric:** `{0:DN}` / `{0:dN}` → `{0:0>N}` (zero-padded)
- **Floating:** `{0:FN}` / `{0:fN}` → `{0:.N}` (decimal places)
- **Scientific:** `{0:EN}` / `{0:eN}` best-effort → `{0:.NE}` / `{0:.Ne}`
- **Explicit rejection:** `X`/`x` (hex), `N`/`n` (thousands), `C`/`c` (currency)

### Integration
- C# mode translates format string first, then uses existing indexed-argument strfmt path
- No breaking changes to `apply_format()` signature
- Error handling propagates translation failures with clear messages
